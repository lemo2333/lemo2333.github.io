NACK — Message Redelivery Support

Designed at-least-once delivery semantics for SMS consumers by introducing negative acknowledgement (NACK) support. Enables consumers to signal processing failures and receive the previous batch for reprocessing.

  • Checkpoint table in DataDemux tracks last non-committed batch offset per consumer
  • On NACK, DataDemux fetches previous batch from RocksDB via gRPC and redelivers to client
  • Changed internalAck timing to stream before Msgs response to support NACK semantics correctly
  • MsgAsyncQueue work item triggers demuxBatch re-dispatch on negative ack
Distributed Messaging gRPC RocksDB At-Least-Once Delivery
